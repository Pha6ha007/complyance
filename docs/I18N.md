# I18N.md — Internationalization Guide

## Overview

Complyance supports 7 languages from launch. Internationalization is a first-class concern, not an afterthought. Every user-facing string, every document, every email must be translatable.

## Languages

| Code | Language | Direction | Status |
|------|----------|-----------|--------|
| `en` | English | LTR | Primary (all strings written here first) |
| `fr` | French | LTR | Translation |
| `de` | German | LTR | Translation |
| `pt` | Portuguese | LTR | Translation |
| `ar` | Arabic | **RTL** | Translation + RTL layout |
| `pl` | Polish | LTR | Translation |
| `it` | Italian | LTR | Translation |

## Technical Setup (next-intl)

### Configuration

```ts
// src/i18n/config.ts
export const locales = ['en', 'fr', 'de', 'pt', 'ar', 'pl', 'it'] as const;
export type Locale = (typeof locales)[number];
export const defaultLocale: Locale = 'en';

export const localeNames: Record<Locale, string> = {
  en: 'English',
  fr: 'Français',
  de: 'Deutsch',
  pt: 'Português',
  ar: 'العربية',
  pl: 'Polski',
  it: 'Italiano',
};

export const rtlLocales: Locale[] = ['ar'];
```

### Middleware

```ts
// src/middleware.ts
import createMiddleware from 'next-intl/middleware';
import { locales, defaultLocale } from '@/i18n/config';

export default createMiddleware({
  locales,
  defaultLocale,
  localeDetection: true, // Detect from Accept-Language header
  localePrefix: 'as-needed' // /en/dashboard → /dashboard for default locale
});

export const config = {
  matcher: ['/((?!api|_next|.*\\..*).*)']
};
```

### Usage in Components

```tsx
// Server component
import { useTranslations } from 'next-intl';

export default function Dashboard() {
  const t = useTranslations('dashboard');
  return (
    <div>
      <h1>{t('title')}</h1>
      <p>{t('complianceScore')}: 72%</p>
    </div>
  );
}

// With interpolation
t('systemsCount', { count: 5 })
// en.json: "systemsCount": "You have {count} AI systems"

// With pluralization
t('gaps', { count: 3 })
// en.json: "gaps": "{count, plural, =0 {No gaps} one {# gap} other {# gaps}}"

// Date formatting
import { useFormatter } from 'next-intl';
const format = useFormatter();
format.dateTime(new Date(), { dateStyle: 'long' });
// English: "March 9, 2026"
// German: "9. März 2026"
// Arabic: "٩ مارس ٢٠٢٦"
```

## RTL Support (Arabic)

### Layout

```tsx
// src/app/[locale]/layout.tsx
import { rtlLocales } from '@/i18n/config';

export default function LocaleLayout({ children, params: { locale } }) {
  const dir = rtlLocales.includes(locale) ? 'rtl' : 'ltr';
  return (
    <html lang={locale} dir={dir}>
      <body>{children}</body>
    </html>
  );
}
```

### Tailwind CSS — Logical Properties

**CRITICAL: Always use logical properties instead of physical ones.**

| ❌ Physical (breaks RTL) | ✅ Logical (works both) |
|-------------------------|----------------------|
| `ml-4` | `ms-4` (margin-start) |
| `mr-4` | `me-4` (margin-end) |
| `pl-4` | `ps-4` (padding-start) |
| `pr-4` | `pe-4` (padding-end) |
| `text-left` | `text-start` |
| `text-right` | `text-end` |
| `left-0` | `start-0` |
| `right-0` | `end-0` |
| `border-l` | `border-s` |
| `border-r` | `border-e` |
| `rounded-l` | `rounded-s` |
| `rounded-r` | `rounded-e` |
| `float-left` | `float-start` |
| `float-right` | `float-end` |

### Icons & Arrows

Arrows and directional icons must flip in RTL:
```tsx
// Use CSS transform or separate RTL icons
<ChevronRight className="rtl:rotate-180" />
```

### Sidebar Navigation

```tsx
// Sidebar should appear on the right in RTL
<aside className="fixed start-0 top-0 w-64">
  {/* This automatically goes to right side in RTL */}
</aside>
```

## Translation Workflow

### Step 1: Write English Strings
All new features start with English strings in `en.json`.

### Step 2: Generate Machine Translations
```bash
pnpm run translate
# Runs scripts/generate-translations.ts
# Uses Claude API to translate en.json → all other locales
# Outputs to respective locale files
```

### Step 3: Review Legal Terms
Legal/regulatory terms must be manually verified against official EU translations (EUR-Lex).

### Step 4: Test RTL
After adding new UI, test Arabic layout:
```bash
# Switch to Arabic in browser
# Check: sidebar position, text alignment, icons, forms
```

## SEO — Locale-Specific Pages

Marketing pages have locale-specific meta tags:
```tsx
export async function generateMetadata({ params: { locale } }) {
  const t = await getTranslations({ locale, namespace: 'marketing.seo' });
  return {
    title: t('title'),
    description: t('description'),
    alternates: {
      languages: Object.fromEntries(
        locales.map(l => [l, `/${l}`])
      )
    }
  };
}
```

`hreflang` tags generated automatically by next-intl for all locale variants.
